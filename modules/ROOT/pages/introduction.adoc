The {plugin-title} plugin helps you maintain clean and valid SCL (Substation Configuration Language) files by identifying and removing elements that have become unreferenced over the lifecycle of an IEC 61850 configuration project.

Over time, as IEDs are added, removed, or reconfigured, SCL files accumulate stale entries â€” DataSets no longer pointed to by any control block, control blocks whose DataSets were deleted, and data type definitions that no longer match any Logical Node.
Leaving these in place does not cause failures, but it increases file size, makes the document harder to reason about, and can trigger false positives in validation tools.

The plugin provides three independent panels, each targeting a distinct category of unreferenced content:

[cols="1,3"]
|===
|Panel |What it removes

|xref:unreferenced-datasets.adoc[Unreferenced Datasets]
|`DataSet` elements not referenced by any `GSEControl`, `SampledValueControl`, `LogControl`, or `ReportControl`

|xref:orphaned-control-blocks.adoc[Orphaned Control Blocks]
|`GSEControl`, `SampledValueControl`, `LogControl`, and `ReportControl` elements whose `datSet` attribute points to a non-existent `DataSet`

|xref:unreferenced-data-types.adoc[Unreferenced Data Types]
|`LNodeType`, `DOType`, `DAType`, and `EnumType` entries in `DataTypeTemplates` not referenced by any Logical Node or other Data Type
|===

== Overview

When you open a document in OpenSCD Shell with this plugin active, each panel immediately analyses the loaded SCL and displays a list of candidates for removal.
You can review each entry, search or filter the list to find what you are looking for, select individual items or all at once, and then delete the selection in a single operation.
All changes are dispatched as edit events and can be undone with the standard undo mechanism.

== When to use this plugin

* After removing IEDs or reconfiguring communication mappings
* Before exporting a final SCL file for commissioning or delivery
* As part of a routine file-hygiene pass in a configuration review workflow
* When a validation tool reports unexpected or excessive data type definitions

== Limitations

* The plugin operates on public SCL elements only â€” private extensions are not examined.
* The plugin does not analyse inter-file references; it works on the currently loaded document.
* Undo is supported through the standard OpenSCD edit history, but very large bulk deletions should be reviewed carefully before committing.
