Installation

1. Try it online

The fastest way to evaluate the plugin is to use the live demo. The demo loads a sample SCD file automatically and activates the plugin. No installation is required.

2. Using with OpenSCD Shell (via URL)

If you are running your own OpenSCD Shell instance, you can load this plugin directly from the continuous integration build:

const editor = document.querySelector('oscd-shell');
const plugins = {
  editor: [
    {
      name: 'Cleanup',
      translations: { de: 'Bereinigung' },
      icon: 'cleaning_services',
      requireDoc: true,
      src: 'https://openpowershift.github.io/oscd-editor-cleanup/oscd-editor-cleanup.js',
    },
  ],
};
editor.plugins = plugins;

The CI bundle always reflects the latest commit on the main branch and may include unreleased changes. For production use, pin to a released npm package (see below).

3. Installing from npm

For production distributions, install the released package:

npm install @omicronenergy/oscd-editor-cleanup

Then register the plugin with the shell:

import OscdEditorCleanup from '@omicronenergy/oscd-editor-cleanup';

const { registry } = document.querySelector('oscd-shell');
registry.define('oscd-editor-cleanup', OscdEditorCleanup);

const editor = document.querySelector('oscd-shell');
const plugins = {
  editor: [
    {
      name: 'Cleanup',
      translations: { de: 'Bereinigung' },
      icon: 'cleaning_services',
      requireDoc: true,
      tagName: 'oscd-editor-cleanup',
    },
  ],
};
editor.plugins = plugins;

3.1. Scoped custom elements

This plugin uses @open-wc/scoped-elements and does not register itself in the global custom element registry. You must use the registry.define pattern shown above, or the plugin will not render.

4. Plugin properties

The plugin accepts the following properties on its element:

Property Type Description

doc

XMLDocument

The loaded SCL document. Set automatically by OpenSCD Shell.

docVersion

unknown

Opaque version token used by the shell to signal document changes and trigger re-renders.

5. Peer dependencies

The plugin is built against and expects the following peer dependencies to be provided by the host application:

{
  "@openscd/scl-lib": "^2.0.0",
  "@openscd/oscd-api": "^0.1.0",
  "@omicronenergy/oscd-ui": "^0.0.11",
  "@omicronenergy/oscd-scl-dialogs": "^0.0.12",
  "lit": "^3.0.0"
